## 项目业务背景

公司内部有数十个甚至上百个代码仓库，沉淀了多年的业务逻辑和技术债务。新员工入职后需要花费数周才能理解系统架构；技术Leader难以从全局视角俯瞰所有项目的健康状况；AI编程助手（如Cursor、Claude Code）缺乏对公司私有代码库的上下文理解，导致辅助效果大打折扣。

我们需要打造一个“企业代码与知识智能平台”，核心能力是：自动拉取并分析公司所有代码仓库，构建代码知识图谱，让员工用自然语言检索项目信息、查看架构关系图，同时对外提供MCP服务供其他AI Agent调用。

## 核心业务模块详细诉求（请按以下结构提取为 PRD 正文）

### 模块一：管理后台（面向 技术负责人 / 数据管理员）

**1. 代码源管理**
- 管理员需要在页面上配置代码仓库连接（Git地址、认证方式（SSH/HTTPS）、分支策略）。
- 系统要能测试连接是否成功。
- 连接后，系统要能自动拉取该仓库的所有基本信息（仓库描述、语言分布、最近提交记录），并展示在列表中。

**2. 项目元数据维护**
- 管理员需要勾选“哪些仓库需要纳入智能检索库”。
- 对于已选中的仓库，管理员可以手动编辑“业务中文名”（例如将 `payment-service` 改为“支付中台服务”）和“业务标签”（例如关联“交易”、“订单”、“退款”等业务关键词）。
- 管理员可以为仓库指定“业务负责人”和“技术负责人”，便于后续问题定位。

**3. 知识库管理**
- 管理员可以上传和维护团队知识文档（如系统设计文档、架构决策记录（ADR）、运维手册），并将其与具体代码仓库或模块进行关联。
- 系统需要支持“培训文档”的一键生成：针对某个仓库或模块，自动提取项目结构、核心功能、API清单、数据表关系，生成一份结构化的培训文档草稿，管理员可编辑后发布。

**4. 架构图与关系图管理**
- 管理员可以查看系统自动生成的架构图（模块依赖关系、服务调用链、数据库表关系），并手动编辑修正。
- 管理员可以将某个版本的架构图“发布为官方版本”，供全员查阅。
- 管理员需要能够管理“架构图模板”（如C4模型模板、分层架构模板），定义不同项目类型使用哪种模板来展示。

**5. 模板管理（非界面模板，指文档/问答模板）**
- 管理员需要有一个界面专门管理“智能问答模板”：当用户问特定类型的问题时，系统按模板格式化输出。
- 示例模板：“当用户问‘XX服务的功能是什么’，输出格式为：服务名称 → 业务职责 → 核心接口 → 依赖的下游服务 → 负责人。”

**6. 监控与预警配置**
- 管理员可以配置代码健康度预警阈值（例如：循环依赖数超过N时预警、单文件行数超过M时预警）。
- 管理员可以配置“架构漂移”预警：当实际代码依赖关系与官方架构图不符时，系统发送通知。


### 模块二：用户前端平台（面向 普通员工 / 开发者）

**1. 智能问答主界面**
- 用户在对话框输入自然语言问题，例如：“支付服务的核心流程是什么？”、“订单表有哪些字段？”、“用户服务和订单服务之间有调用关系吗？”。
- 系统需要识别问题类型（架构类、代码类、文档类、人员类），并路由到对应的知识来源。

**2. 模板推荐与快捷套用**
- 当用户输入问题时，系统异步匹配现有的问答模板。
- 如果匹配度较高，对话框上方以卡片形式提示：“检测到相似问题模板，是否按标准格式查看？”
- 用户点击“套用”则按模板格式输出，点击“忽略”则走自由生成模式。

**3. 流式交互与打断**
- 系统回答必须像 ChatGPT 一样逐字流式推送。
- 推送过程要分阶段展示状态：“正在理解问题...”、“正在检索代码图谱...”、“正在生成回答...”、“正在整理文档...”等。
- 中断能力：在系统回复过程中，发送按钮变为“停止生成”按钮，用户点击后生成立即终止，已生成内容保留并标记“已中断”。

**4. 对话上下文与连续追问**
- 用户在任意历史对话气泡下，可以输入追加条件（例如“那它的下游依赖有哪些？”），系统必须能识别这是对上一轮对话的延续。
- 系统需要维护当前会话的“上下文锚点”（例如当前在讨论哪个服务、哪个模块），以便连续追问时精准定位。

**5. 架构图与关系图浏览**
- 用户可以在对话中请求“画一下XX服务的架构图”，系统返回一张可交互的架构图（支持缩放、拖拽、点击节点查看详情）。
- 用户也可以独立进入“架构图浏览”模式，按项目/系统查看官方发布的架构图。
- 点击架构图中的节点（如一个微服务），可以查看该服务的详细信息（负责人、技术栈、最近变更、相关文档）。

**6. 历史会话管理**
- 左侧边栏展示历史会话列表，按日期倒序排列。
- 用户可以重命名会话标题、删除会话。
- 用户可以将会话“收藏”或“标记为重要”，便于后续回顾。

**7. 代码检索（进阶能力）**
- 支持语义检索：“查找所有处理订单状态回滚的代码”，系统返回相关代码片段及其所属文件。
- 支持符号检索：输入函数名/类名，系统定位到定义位置并展示调用关系。


### 模块三：监控平台（面向 技术Leader / 架构师 / 运维）

**1. 代码健康度全局看板**
- 展示所有已纳管项目的“健康度评分”（综合指标：代码复杂度、重复率、循环依赖数、测试覆盖率等）。
- 按团队/项目/语言维度下钻查看详情。
- 高亮展示“风险项目”（评分低于阈值），点击可查看具体风险项列表。

**2. 架构合规性看板**
- 展示“官方架构图”与“实际代码结构”的差异对比。
- 出现架构漂移时，看板高亮预警，并标注具体漂移位置（例如：A服务实际调用了B服务，但架构图中未声明）。

**3. 知识库质量看板**
- 展示各项目的文档覆盖率（有多少模块有对应的知识文档）。
- 展示培训文档的生成与发布数量趋势。
- 展示用户对问答结果的点赞/点踩统计，帮助识别低质量知识源。

**4. MCP服务调用监控**
- 展示MCP服务被外部AI Agent（如Cursor、Claude Code）调用的次数、频率、调用方来源。
- 展示MCP服务的响应时间、成功率、热门查询词云。
- 当调用量异常波动或错误率突增时，看板预警。

**5. 索引与更新状态看板**
- 展示各代码仓库的索引状态（已索引/索引中/索引失败）。
- 展示近24小时因代码变更触发的增量更新记录。
- 展示“索引失败”的仓库列表及失败原因摘要。
