## 一、管理端（Admin Dashboard）- 面向技术负责人/管理员

**风格定位**：极简主义 + 功能主义，偏重操作效率。
**色彩**：以深色侧边栏（#1E293B）搭配浅色主内容区（#F8FAFC），主色调使用冷静的蓝色（#3B82F6）。
**布局**：左侧固定导航（包含：代码源管理、项目元数据、知识库管理、架构图管理、模板管理、监控预警配置），右侧内容区包含数据统计卡片（仓库总数、已索引数、文档覆盖率、待处理预警）、表格（仓库列表、索引任务队列）和表单（新增仓库配置、编辑元数据）。
**排版**：使用系统无衬线字体（Inter 或 SF Pro），信息层级清晰，字号偏小以适应密集数据。
**氛围**：专业、可信赖、高信息密度，但有序。

## 二、用户端（User Portal）- 面向普通员工/开发者

**风格定位**：温暖 + 品牌化，注重情感连接。
**色彩**：主色调用暖橙色（#F97316）或品牌色，搭配柔和米白背景（#FFF7ED），点缀以浅灰色。
**布局**：顶部导航（Logo + 搜索框 + 用户头像），内容区域以卡片式布局展示推荐知识或快捷入口，留白充足，大标题为主。左侧保留历史会话列表（可折叠）。
**排版**：标题使用衬线字体（如 Merriweather 或 Playfair Display），正文使用无衬线字体，字号偏大。
**氛围**：友好、亲切、有活力，仿佛一个值得信赖的协作伙伴。

## 三、监控端（Monitoring Dashboard）- 面向技术Leader/架构师

**风格定位**：数据驱动 + 科技感，强调实时性。
**色彩**：深色背景（#0F172A）搭配霓虹蓝绿（#06B6D4）和紫色（#8B5CF6）作为强调色，关键指标使用亮色（如红色预警、绿色健康）。
**布局**：全屏看板，顶部为关键指标（KPI）卡片（健康度平均分、架构合规率、MCP调用总次数、今日索引变更数），中间区域为多个图表（折线图展示健康度趋势、柱状图展示各项目风险分布、环形图展示文档覆盖率），底部为实时预警日志列表。
**排版**：使用等宽字体（如 JetBrains Mono）显示数据，数字要大而醒目。
**氛围**：未来感、冷静、数据权威感，仿佛一个大型监控指挥中心。

## 额外要求（三端通用）
1. 三个页面需要保持品牌一致性：页面标题栏统一显示“灵镜 (LingPrism)”标识。
2. 底部统一添加“© 2026 NL Hermes”的页脚。
3. 请为每个端生成独立的 HTML 文件，确保可以直接在浏览器中运行，所有页面都可以点击跳转。
4. 文件命名规则：`admin.html`、`user.html`、`monitor.html`，统一存放在 `docs/mockup/` 目录下。
5. 图表部分可以使用 Chart.js CDN 或纯 CSS 模拟，确保视觉上接近真实图表即可。
6. 所有交互按钮、导航菜单只需展示 UI 状态，无需绑定真实后端逻辑，但鼠标悬停（hover）和点击反馈（active）的 CSS 动效需要体现。

## 设计要点

**管理端** — 深色侧边栏 `#1E293B` + 浅色内容区，Inter 字体，高密度表格与表单；左侧 6 项导航可切换内容区，含仓库列表、索引队列、新增配置表单。

**用户端** — 暖橙 `#F97316` + 米白 `#FFF7ED`，Merriweather 标题 + Source Sans 3 正文；顶部搜索栏、可折叠历史会话、PRD 中的支付服务问答场景（模板推荐 + 上下文追问）。

- 底部统一「© 2026 NL Hermes」
