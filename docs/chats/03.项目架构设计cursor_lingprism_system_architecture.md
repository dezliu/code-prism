## 新增：§4.7 LLM 多厂商适配层

在 `services/ai-worker/infrastructure/llm/` 增加统一适配层，**LangChain Chains 不直接实例化 LLM**，全部通过 factory 获取：

```
infrastructure/llm/
├── config.py      # 解析 env / 管理后台 DB 配置
├── factory.py     # create_chat_model(scene) / create_embedding_model()
├── providers/     # deepseek / qwen / zhipu / openai
└── streaming.py   # 统一流式输出 + 中断（PRD 4.2.3）
```

## 支持的厂商

| Provider | 厂商 | Chat 默认 | Embedding 默认 |
|----------|------|-----------|----------------|
| `deepseek` | DeepSeek | `deepseek-chat` | `deepseek-embedding` |
| `qwen` | 通义千问 | `qwen-max` | `text-embedding-v3` |
| `zhipu` | 智谱 GLM | `glm-4-plus` | `embedding-3` |
| `openai` | OpenAI（可选） | `gpt-4o` | `text-embedding-3-small` |

三家国产模型均走 **OpenAI 兼容 API**（`/v1/chat/completions` + `/v1/embeddings`），用 LangChain `ChatOpenAI` + 自定义 `base_url` 接入，切换厂商只需改 env，无需改业务代码。

## 配置方式

1. **环境变量（Phase 0）**：`LLM_PROVIDER=deepseek`，各厂商独立 `*_API_KEY` / `*_BASE_URL` / `*_MODEL`
2. **场景分流（可选）**：`LLM_SCENE_QA_MODEL`、`LLM_SCENE_INTENT_MODEL` 等，意图识别用小模型、问答用大模型
3. **Chat / Embedding 可异厂商**：例如 Chat 用 DeepSeek、Embedding 用千问（`EMBEDDING_PROVIDER=qwen`）
4. **管理后台（Phase 1+）**：`apps/admin/settings/llm` 页面，API Key 加密存 MySQL，支持连通性测试与热更新

## 其他约束

- **MCP `ask_question`** 复用 ai-worker 的 factory，不在 `services/mcp` 重复接 LLM
- **Langfuse** 按 `provider + model` 分维度埋点，便于监控各厂商 latency / token
- **Embedding 换厂商** 可能改变向量维度 → Qdrant 需分 collection 或重建索引（已列入待确认项）

| 批次 | Todo | 策略 | 预估 |
|------|------|------|------|
| **Batch 0 地基** | `env-templates` → `scaffold-monorepo` → `infra-docker`（仅数据层） | 串行 | 0.5~1 天 |
| **Batch 1 骨架** | `scaffold-api` / `core` / `indexer` / `mcp` / `ai-worker` | **5 轨并行** | 1~2 天 |

| **Batch 3 用户闭环** | `local-auth-minimal` → `sse-chat-stream` | auth 完成后 SSE | 1~2 天 |
| **Batch 4 集成** | `infra-docker`（Nginx + Dockerfile + README） | 等入口稳定后 | 0.5~1 天 |
| **Batch 5 P0 业务** | `phase1-p0` 再拆 A/B/C | 最后 | 2~4 周 |

## 关键决策

**1. `env-templates` 必须最先**  
避免各服务各自定义环境变量，后期统一成本高。

**2. Batch 1 是并行窗口最大的阶段**  
api / core / indexer / mcp / ai-worker 互不写库，可同时推进；建议按语言拆 2~3 个 PR 审查。

**3. `infra-docker` 拆两阶段**  
- Batch 0：MySQL、Redis、Neo4j、Qdrant、OpenSearch（让 Batch 1 能连库）  
- Batch 4：Dockerfile + Nginx（等服务端口定稿后再写，减少返工）

**4. Batch 3 是 Phase 0 里程碑**  
验收标准：登录 → SSE 流式问答 → 停止生成。完成后即可对外演示。

**5. `phase1-p0` 内再拆 3 子批**  
- **P0-A**：代码源 + 索引流水线 + 索引看板  
- **P0-B**：智能问答 RAG + 会话  
- **P0-C**：架构图 + 健康/合规看板
