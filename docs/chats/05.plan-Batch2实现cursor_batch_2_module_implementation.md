### 修改了哪些文件

**ai-worker（测试）**

- `internal/infrastructure/qdrant/collections_test.go`
- `internal/infrastructure/config/config_test.go` — 新增推导逻辑测试

### 改了什么行为

| 能力 | 行为 |
|------|------|
| LLM 切换 | `LLM_PROVIDER` / `EMBEDDING_PROVIDER` 在 4 厂商间切换，默认智谱 |
| 场景模型 | `LLM_SCENE_*_MODEL` 覆盖 qa/intent/doc_gen；intent 默认 `glm-4-flash` |
| Factory | 有 API Key → 真实 LangChain 模型；无 Key → Placeholder（CI 友好） |
| Collection 命名 | 两侧统一：`lingprism_v1_{provider}_{dim}`，默认 `lingprism_v1_zhipu_1024` |
| Qdrant 写入 | `ensure_collection()` 在 collection 不存在时自动创建 |

### 为什么这样改

- **OpenAI 兼容协议**：DeepSeek/千问/智谱均走 `ChatOpenAI` + `base_url`，切换成本低
- **env 优先**：Batch 2 不做 DB 热加载（Batch 3+ 扩展）
- **读写同名**：ai-worker 写入与 core 检索共用同一命名函数，避免 Phase 1 索引返工
- **Placeholder 回退**：无 API Key 时仍可跑单元测试，不依赖外网
