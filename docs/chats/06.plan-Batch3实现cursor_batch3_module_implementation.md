### 改了什么行为

| 能力 | 行为 |
|------|------|
| 登录 | GraphQL `login(email, password)` → JWT + 用户信息；`me` 查询当前用户 |
| 三端登录 | `/login` 页 + localStorage token + Apollo Bearer 注入 |
| SSE 问答 | JWT 保护 `POST /api/chat/stream` → 流式 `status/token/done` 事件 |
| 停止生成 | `POST /api/chat/stop` + Redis 取消令牌，≤2s 中断语义 |
| 用户平台 | 登录后可发送问题、实时流式显示、点击停止 |

### 为什么这样改

- **auth 在 api 层**：对齐计划，MySQL `users` 表由 Node GraphQL 写入，JWT 无状态会话
- **SSE 走 api 网关**：前端只访问 api；LLM 流式委托 ai-worker，避免 Node 重复接 LLM
- **取消令牌复用 Batch 2 `streaming.py`**：api 与 ai-worker 共用 Redis key 前缀
- **测试用 MemoryStreamCancelStore / MockAiWorker**：CI 不依赖 Redis / 外网
