### 为什么这样改

- **MySQL 写权限边界**：Node 写业务 CRUD，Go 写索引产出（`index_jobs`、`health_scores` 等）
- **api → core 走 HTTP 内部 API**：避免 Batch 5 引入 protoc 代码生成，契约清晰、可独立测试
- **qa_router 替代 simple_qa**：在 Batch 3 SSE 骨架上叠加 RAG，不重复实现 LLM 适配层
- **测试用 PassthroughChatOrchestrator**：SSE 集成测试不依赖 MySQL

**根因**：MySQL 严格模式下 JSON 列不能带默认值；首次失败后又留下了半完成表结构。
