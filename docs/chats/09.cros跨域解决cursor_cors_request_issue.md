## 变更摘要

### 修改了哪些文件
- `services/api/src/infrastructure/http/cors.ts` — 新增 `expandOriginAliases`，自动镜像 `localhost` ↔ `127.0.0.1`；默认源增加 `8080`
- `services/api/src/config.ts` — 统一使用 `parseCorsOrigins`
- `services/api/src/infrastructure/http/server.test.ts` — 新增 `127.0.0.1:3000` 预检测试用例

### 改了什么行为
浏览器无论用 `http://localhost:3000` 还是 `http://127.0.0.1:3000` 访问前端，GraphQL 跨域预检都能通过。

### 为什么这样改
CORS 白名单只配置了 `localhost`，而浏览器地址栏用 `127.0.0.1` 时 Origin 不同，会被拦截。`Status code: (null)` 也可能是因为 API 未启动导致连接失败，需同时确认 `services/api` 在 4000 端口运行。

- Docker/Nginx 场景下若前端源为 `user.localhost:8080` 等，需在 `.env` 的 `CORS_ORIGINS` 中显式配置
