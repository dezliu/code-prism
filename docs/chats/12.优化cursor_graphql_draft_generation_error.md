### 改了什么行为

**1. 架构图「生成草稿」报错**

根因：`graphData` 是对象类型 `GraphData!`，GraphQL 要求必须选择子字段，不能只写 `{ graphData }`。

已改为与用户端架构页一致的查询：

```graphql
graphData { nodes { id label type } edges { id source target label } }
```

**2. 数据源管理支持删除**

- 仓库列表操作列新增红色「删除」按钮
- 点击后弹出确认框，说明会一并删除元数据、架构图、索引记录
- 后端新增 `deleteRepo(repoId: ID!): Boolean!` mutation
- 数据库层通过 `ON DELETE CASCADE` 级联清理关联数据

### 为什么这样改

- GraphQL 校验错误是前端查询写法问题，与用户端已有正确写法对齐即可
- 删除能力沿用项目中 `deleteQaTemplate` 的模式（Repository → UseCase → GraphQL → 管理端确认弹窗），改动范围最小
