## 变更摘要

管理端「架构图」模块已改为与知识库一致的 **列表优先** 交互：进入即见列表，通过「添加架构图」选择代码库生成草稿。

### 修改了哪些文件

- `apps/admin/src/components/panels/ArchitecturePanel.tsx` — 重构为列表 + 添加弹窗 + 编辑弹窗
- `services/api/src/graphql/schema/architecture.graphql.ts` — 新增 `adminArchitectures`、`architectureDraft` 查询及摘要类型
- `services/api/src/application/architecture/architecture.use-cases.ts` — 新增 `ListAdminArchitecturesUseCase`
- `services/api/src/infrastructure/db/repositories/monitor.repository.ts` — 新增 `listArchitectureRepoIds`
- `services/api/src/graphql/resolvers/index.ts`、`context-factory.ts` — 注册新查询

### 改了什么行为

1. **默认展示列表**：显示各代码库的草稿状态、官方版版本与节点数
2. **添加流程**：点击「添加架构图」→ 选择尚未添加的代码库 → 自动生成草稿并打开编辑弹窗
3. **编辑弹窗**：预览架构图、重新生成、填写版本说明后发布官方版（无草稿时发布按钮禁用）

### 为什么这样改

对齐 PRD 4.1.4 与知识库面板交互模式：先列表治理，再按需生成/编辑，避免一进来就是空白生成页。

### 有哪些假设

- 每个代码库在列表中占一行；已添加的代码库不会出现在「添加」下拉中，需通过「重新生成」更新

### 还剩哪些风险或未验证项

- 需在本地启动管理端验证 GraphQL 新接口与完整流程
- 发布后草稿变为官方版，需再次「重新生成」才有新草稿可发布——符合当前后端逻辑
