### 改了什么行为

1. **生成流程**：同步 Git（本地优先，有远端新提交才 pull）→ 架构专用上下文 → LLM 分析笔记 → 生成 `GraphData` JSON → 校验失败则带错误信息重试（最多 3 次）→ 写入 `graph_snapshots` 草稿
2. **双执行模式**：编辑弹窗 SSE 流式（显示 phase 进度）；列表页「后台生成」+ 任务列表 Modal
3. **数据契约**：输出严格符合 `{ nodes: [{id, label, type}], edges: [{id, source, target, label?}] }`，`type` 仅 `service|module|database`

### 为什么这样改

- 编排放在 API 层（与文档生成一致），Core 只管 Git/上下文，ai-worker 只管 LLM
- 校验重试在 API 侧闭环，避免无效 JSON 入库
- 架构上下文单独筛选（docker/k8s/入口/路由/schema），比通用 `BuildDocContext` 更贴合架构图

- 超大 monorepo 上下文会截断（80K 字符/仓库）
- Go 旧端点 `/internal/architecture/{repoId}/generate-draft` 仍保留但返回 deprecated 错误；API 已不再调用
- 发布后 Neo4j 同步未在本 PR 触发
