支付服务的核心流程是什么？
模板推荐 匹配度 92%

检测到相似问题模板「服务功能介绍」，是否按标准格式查看？
输出格式：服务名称 → 业务职责 → 核心接口 → 依赖的下游服务 → 负责人

灵镜
支付中台服务的核心流程如下：

1. 收单：接收上游业务系统的支付请求，校验订单信息与金额
2. 路由：根据支付渠道策略选择最优通道（微信/支付宝/银联）
3. 执行：调用渠道 SDK 发起扣款，维护幂等键防止重复支付
4. 回调：接收渠道异步通知，更新订单状态并通知下游订单服务
5. 对账：T+1 日与渠道账单核对，差异进入人工处理队列 📄 来源：支付服务 ADR-003
你
那它的下游依赖有哪些？
灵镜
基于上下文，「它」指代支付中台服务，其下游依赖包括：

• order-platform — 接收支付结果回调，更新订单状态
• notification-hub — 发送支付成功/失败通知
• reconciliation-service — 对账差异处理
• risk-engine — 实时风控校验（同步调用） ◇ 来源：2026-Q2 官方架构图
